# 快速开始

## 代码环境

本项目主要包含两个部分：本体代码与人物卡文件。本体代码由Python写成，由pyinstaller打包制作为exe文件。

本项目需要一个独立的运行环境，请保证目录为全英文目录。

将写好的人物卡拖入根目录下characters文件夹内，代码启动时就会自动识别并导入人物卡。

## 代码运行逻辑

对于整个代码的运行，遵循如下的流程与运行逻辑。

首先，代码读取所有位于characters文件夹内的txt文件，并以其文件名为`truly_id`，建立所有人物卡的对象。

然后需要使用者输入**系统代码**（详见代码文档），配置好战场环境后开始战斗。

在战斗刚刚开始，即进入战斗**主线程**时，会创立一个名为`listen`的监听对象。它会遍历场上所有角色的被动技，并将其加入至**ocses（监听列表）**中。

在将所有被动技加入至`ocses`列表后，主线程会立即检测有无需要触发的**ocs（监听对象）**。即，在此处实现了一些被动技的效果（如饰品的“本局战斗内敏捷+5”）。

然后主线程会进入轮次的循环中，在每个关键节点，主动调用`listen`的`check`功能，检测当前节点有无需要触发的`ocs`。如果有则执行，并将其从`ocses`中删除。

而技能的释放，则是将对应`skill`对象的`effect`内容，转化为`ocs`对象，并加入至`ocses`列表的底端。然后立即发起一次`listen.check('immediate')`，从而实现技能的释放。

在战斗的过程中，使用者也可以随时输入系统代码，从而实现系统层面的一些功能。

## 人物卡的结构

人物卡为txt格式，请确保编码为utf-8。Urfis对空格、缩进要求均不严格，但推荐使用缩进以增强可读性。

在代码最外层，由数个“基本对象”数据结构构成。“基本对象”的格式如下：

```
name:[
	key1 = value
	key2:
		Urfis codes
	;
]
```

其中，每个具体的“基本对象”将会在下方一一介绍。

需要注意的是，等号后面的value值不能为空白。虽然有些属性为可选属性，但建议全部写上。而且一旦写上，等号后就不能留空。

除`item`、`skill`对象外，其余对象最多只能出现一遍，其中`basic`更是必须出现。而对于对象内的属性，则每种属性均只能出现一遍。

## Urfis代码的结构

在`effect`属性内，可以通过写Urfis代码，从而实现数据的获取与处理。

Urfis有两类代码：方法与函数。方法的作用是获取一个“对象”的某个属性，函数的作用则是对一个对象进行处理。

方法遵循如下结构：`ta.ac`。函数则遵循如下结构：`ta.ac(pa)`。其中`ac`为方法/函数名，必需。`ta`为方法/函数的目标，可选。`pa`为函数的参数，可选。

例如：要实现“对伤害来源的角色造成10点魔法火属性伤害”这个效果，我们需要将其拆解。首先，找出句子的“宾语”或“动词”，即“造成伤害”，对应了`damage()`函数，这便是`ac`。

`damage()`需要一个目标与参数，目标`ta`很明显为“伤害来源的角色”，参数`pa`即其他的要求，即具体的伤害值（10）、伤害类型（魔法）、伤害属性（火属性）。

通过查阅说明文档，我们得知标准写法为`damage(10, m, huo)`，但我们并没有找到“伤害来源的角色”这个现成的目标。因此，我们就需要继续拆解这个`ta`。

“伤害来源的角色”可以分为“伤害”和“来源角色”，而通过查阅说明文档，我们得知伤害属于`Damage`对象，而`w_damaged`时机中又提供了`d.`目标。因此我们可以继续查阅`Damage`对象的属性，发现其中有`from`这个属性，正好对应了“伤害来源的角色”。

因此，`d.from`便是一个方法，可以简单将其理解为没有参数的函数。他提供（返回）`Character`对象，正好我们提供查阅说明文档发现，`damage`的`ta`需要一个`Character`对象，因此就可以直接连起来写了。

即：最终的代码为`d.from.damage(10, m, huo)`。从中我们可以发现，方法可以无限叠加，函数也是同理。只要说明文档中提及这个函数有返回值，就可以直接将其作为新的`ta`，放在别的函数的前面。

## basic对象

basic对象容纳了该人物卡的所有基本属性，全部为**必须属性**，具体属性如下：

### name

该角色的姓名，如 `name = 赤瞳刺角兽`

###	strength

该角色的力量（体质），如 `strength = 25`

### agility

该角色的敏捷（精巧），如 `agility = 15`

### wisdom

该角色的智慧（博学），如 `wisdom = 10`

### spirit

该角色的意志（精力），如 `spirit = 19`

### charm

该角色的魅力（交涉），如 `charm = 0`

### luck

该角色的幸运，如 `luck = 17`

### fund

该角色的资金，如 `fund = 0`

###	hp

该角色登场时的生命值，如 `hp = 300`

###	hp_max

该角色的最大生命值，如 `hp_max = 350`

###	ap

该角色登场时的ap值，如 `ap = 8`

###	ap_max

该角色的最大ap值（ap上限），如 `ap_max = 8`

###	level

该角色的阶数，如 `level = 4`

###	dodge

该角色的额外闪避加成，如 `dodge = 8`

###	block

该角色的额外格挡加成，如 `block = 4`

## 其他对象共用的属性

除 `basic` 对象外，还有 `main_weapon`、`secondary_weapon`、`armor`、`accessory`、`item`和`skill` 对象。
以上对象均有如下属性，除特别标注外均为**必须属性**：

### name

该对象的名称，如 `name = 炽烈甲`

###	type

该对象的类型，如 `type = 药物`

需要注意的是，对于`skill`以外的对象来说，`type`的作用只是储存着文字信息，并没有其他功能，因此为可选属性。

在effect代码中，可以通过判断`type`来判断攻击方式，如`if (equal(item.type, 物品类型)){}`，但这种比较是单纯文字上的比对。如果人物卡中 `type` 写的是“草帽”，而代码中比对的目标是“帽子”，那么也会返回“两者不相等”的结果。

而对于`skill`对象来说，`type`则只有两种可选项：`pr`或者`pa`。`type = pr`（proactive）意为当前技能为主动技，`type = pa`（passive）意味当前技能为被动技。

被动技与主动技的不同有两点，其一为“被动技无法在行动阶段被主动释放”，其二为“被动技会在战斗开始时主动加入监听列表”。除此以外，两者没有任何区别。被动技的监听周期也只有一次，与主动技相同。

### level

该对象的等阶，如 `level = 2`

### from

该对象的来源，如 `from = 捡来的`

该属性与`type`类似，即使对于`skill`对象，也都只提供储存文字信息的作用，为可选属性。

### desc

该对象的说明，如 `desc = 备注`

该属性与`type`类似，即使对于`skill`对象，也都只提供储存文字信息的作用，为可选属性。

### effect

该对象的效果，也是唯一一个可以留空的属性，如：

```
effect:
	Urfis code
;

effect:
;
```

需要特别注意的是，主武器（`main_weapon`）的`effect`即普攻时触发的效果。

代码规定`main_weapon`、`secondary_weapon`、`item`和将`type = pr`的`skill`对象的`effect`，均为主动技效果。与之相对的，`armor`、`accessory`和将`type = pa`的`skill`的`effect`，均为被动技效果。

## main_weapon与secondary_weapon对象

`main_weapon`负责定义角色的主武器，`secondary_weapon`负责定义角色的副武器，两者具有相同的格式。

### method

该武器的攻击方式，如 `method = 刺`

该属性与`desc`类似，仅提供文字储存的功能，为可选属性。

### ap

该武器攻击所消耗的ap，如 `ap = 3`

使用武器攻击的情况即角色的普攻，此处的ap值即角色普攻时所消耗的ap值。

###	from

该武器的来源，如 `from = 垃圾桶里捡来的`

该属性与`desc`类似，仅提供文字储存的功能，为可选属性。

## armor对象

负责定义角色的护甲。

###	p_a

该护甲所拥有的物理护甲（physical），如 `p_a = 8`

受到伤害时，会优先扣除护盾，其次抵消护甲值，然后再结算其对hp的伤害。

###	m_a

该护甲所拥有的魔法护甲（magic），如 `m_a = 8`

受到伤害时，会优先扣除护盾，其次抵消护甲值，然后再结算其对hp的伤害。

## accessory对象

负责定义角色的饰品，没有特殊的属性。

## item对象

负责定义玩家的物品，可以在人物卡中出现多次。

### quantity

该物品的数量，如`quantity = 2`

需要注意的是，当玩家将某件物品给别的玩家后，会以“一件”的数目给出。同时给出的物品，由于无法判断是否与已有的物品相同，因此均为判定为不同类型且数量为一的物品。

## skill对象

负责定义玩家的技能，可以在人物卡中出现多次。

### attr

该技能的属性，如`attr = zhiyu`，可选的值如下：

|值|含义|值|含义|
|----|----|----|----|
|jin|金属性魔法|zhiyu|治愈魔法|    
|mu|木属性魔法|shikong|时空魔法|
|shui|水属性魔法|xuemo|血魔魔法|
|huo|火属性魔法|xinling|心灵魔法|
|tu|土属性魔法|guangming|光明魔法|
|feng|风属性魔法|heian|黑暗魔法|
|lei|雷属性魔法|ying|影魔法|
|bing|冰属性魔法|zhoushu|咒术|
|zhenqi|真气魔法|wuji|武技|
|zhaohuan|召唤魔法|qita|其他|

需要注意的是，这里只是定义了技能的属性，而且理论上可以自定义属性。

在造成伤害时，还会额外要求提供伤害的属性，而这两个属性可以不同（如使用真气魔法造成了风属性伤害）。

### ap

发动该技能所消耗的ap值，如`ap = 3`

被动技也需要定义其ap值，但应当定为0
